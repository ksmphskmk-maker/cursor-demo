# PR 준비 점검

변경된 파일들을 PR 올리기 전 점검해줘.

## 점검 절차

1. `npm test`를 실행하고 결과를 요약한다.
2. 테스트가 통과하면 변경 사항을 한국어 3줄로 요약한다.
3. Conventional Commits 형식(`feat:`, `fix:`, `chore:`, `refactor:`)으로 커밋 메시지 제목 1줄 + 본문을 제안한다.
4. 마지막에 리뷰어가 특히 봐야 할 부분을 짚어준다.

## 출력 형식

아래 순서와 형식으로 답변한다.

### 1. 테스트 결과

- 통과/실패 여부
- 실패 시 실패한 테스트와 원인 요약

### 2. 변경 사항 요약 (한국어 3줄)

- 1줄: 무엇을 바꿨는지
- 2줄: 왜 바꿨는지
- 3줄: 영향 범위 또는 주의할 점

### 3. 커밋 메시지 제안

```
<type>: <제목>

<본문>
```

### 4. 리뷰 포인트

- 리뷰어가 특히 확인해야 할 파일, 로직, 엣지 케이스를 bullet로 정리한다.
